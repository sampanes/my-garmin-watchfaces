# Ink Sandbox

This is a fast local playground for testing generative ink / wash ideas outside Garmin.

Design goals:

- fixed `416x416` canvas to stay close to the Forerunner 265 screen
- immediate-mode 2D drawing
- alpha compositing
- simple procedural ridge / stamp / mist experiments
- fast browser iteration

## Files

- `index.html`
- `style.css`
- `main.js`
- `package.json`

## Recommended workflow

If Node is installed:

```bash
npm install
npm run dev
```

Then open the local URL Vite prints, usually:

```text
http://localhost:5173
```

This gives you live reload on save.

## If you do not want Vite

You can also serve this as a plain static folder:

```bash
python -m http.server 4173
```

Then open:

```text
http://localhost:4173/playgrounds/ink-sandbox/
```

That path will not live reload automatically, but it is still useful.

## Current rendering approach

The current `main.js` uses a fill-first strategy:

- mountains are filled shapes, not outlines
- the ridge is defined by where the fill stops, never by a drawn line
- each layer uses a rapid gradient falloff: most ink lives in the top ~20% of the shape, then fades aggressively toward transparent
- `shadowBlur` with a slight upward offset fakes wet-ink bleed at the ridge without extra computation
- a second diluted fill pass over the same shape simulates ink thinning as the brush runs dry

The HR data drives the middle layer directly. Smoothing window size is the main control:
- window ~65 → 3–4 gentle humps (distant range character)
- window ~24 → 8–10 peaks (main mountain range character)
- window ~80 → 2–3 humps (simple foreground silhouette)

### Noise subsystem (current)

Mountain profiles are generated by summing HR-driven Gaussian peaks (base shape) with three scales of noise on top:

- **SimplexNoise 2D** — seeded 2D simplex replaces the old 1D value noise table. More isotropic; no directional grain artifacts.
- **Fractal Brownian Motion (fBm)** — `sum(amplitude^i * noise(frequency^i * pos))` over N octaves, with explicit lacunarity and gain controls. Replaces `oNoise`. Used at mid and high frequencies for hill-scale and grit-scale variation.
- **Domain warping** — nested fBm calls: `f(p + fbm(p + fbm(p)))`. Distorts sample coordinates before the final noise read, producing "folded rock" quality at ridge edges — organic creases rather than smooth bumps.
- **Turbulence** — absolute-value fBm. Produces a veiny, craggy texture suited to rock face detail.

### Brush mark subsystem (current)

- **Pressure-varying cun strokes** — each stroke is drawn segment-by-segment with lineWidth driven by a bell-curve formula: `sin(πt)^0.6 * (1 + 0.3 * noiseModulation)`. Thin at entry and exit, full at midpoint — matching traditional calligraphic pressure character.
- **Dry brush texture pass** — near-horizontal dashed marks inside the mountain body, simulating bristle skip on washi.
- **Flying white** — low-opacity dashed marks at the ridge.

### Tree subsystem (current)

- **Recursive branch generator** — `generateBranch()` recurses to depth 3 with noise-guided angle deviation and pressure-varying lineWidth per segment.
- **Foliage clusters** — simplex-perturbed filled polygons at terminal branches; irregular blob outline rather than circles.

### Ink diffusion (Garmin-compatible)

Pixel-level anisotropic bleeding (via `getImageData`/`putImageData`) is not implemented here — too slow for hot-reload iteration and has no equivalent on Garmin's `Dc`. The `bleedEdge()` function approximates diffusion with radial-gradient micro-blobs scattered along the ridge, which translates directly to `dc.fillCircle` calls on Garmin.

## What to experiment with next

- **Domain warp strength** (currently 0.3–0.4) — higher values produce more folded, geological ridgelines; lower values approach smooth fBm terrain
- **fBm octave count** — 6 octaves for far mountains, drop to 3–4 for faster foreground; pushing to 8 adds micro-grit but costs perf
- **Turbulence weight** relative to domain warp — more turbulence = veinier rock face; less = cleaner ink wash read
- **Pressure curve exponent** (currently 0.6) — lower = faster pressure buildup, rounder marks; higher = more pointed calligraphic entry/exit
- **Tree depth and branching spread** — depth 3 gives sparse silhouette pines; depth 4–5 with tighter spread produces denser foliage masses
- **Foliage cluster radius and blob count** — currently 3–7 blobs at 0.25–0.65 × branch-length radius; more blobs at lower alpha looks more like ink wash foliage
- **Gradient falloff curve** — steeper initial drop (opacity gone by 10% depth rather than 38%) pushes toward sumi-e references where ink pools only at the crest
- **Y range positioning** for each layer — affects sky visible above far mountains
- **Color temperature** of distant vs near layers — far layers may benefit from a faint cool blue-grey shift
- **Mist band width and opacity** — a second pass at a slightly shifted Y improves the dissolving-edge quality

## What has failed

**Stamp clusters on peaks** (first iteration): placing brush stamps at every local maximum, at 2px point spacing, produced a curtain of vertical columns. The stamps need to be placed at major peaks only, or not at all for the mid-layer.

**Explicit ridge line drawn over the fill** (first iteration): drawing a stroke along the top of the mountain after filling it produces an EKG-on-mountains effect. The ridge character should come from the fill shape and gradient only.

**Small smoothing window** (first iteration, window=9): with 180–220 HR points mapped to 416px, a 9-point smooth still left ~50 micro-peaks per layer — spiky terrain instead of mountains. A window of 20+ is the minimum for mountain-shaped output.

## Rule of thumb

The mountain IS the fill. Shape and gradient carry the whole weight.

1. choose a smoothed profile that reads as terrain, not signal
2. fill it with a gradient that fades fast — most ink at the ridge, nearly gone by midpoint
3. let shadowBlur do the wet-edge work
4. add mist between layers, not as decoration but as concealment
5. add no more than 2–3 calligraphic marks total; everything else is subtraction
